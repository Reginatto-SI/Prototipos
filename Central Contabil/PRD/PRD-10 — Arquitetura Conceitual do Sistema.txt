# PRD-10 — Arquitetura Conceitual do Sistema
# Central Contábil

---

## 1. Objetivo

Definir a estrutura conceitual do sistema, garantindo:

- organização dos dados
- separação clara de responsabilidades
- base consistente para desenvolvimento
- alinhamento entre todos os módulos

Este documento não define comportamento (PRD), mas sim a estrutura do sistema.

---

## 2. Princípio Arquitetural

O sistema é baseado no conceito:

👉 Cliente como HUB central

Todos os dados operacionais orbitam em torno do cliente.

---

## 3. Estrutura Geral

```

Office
├── Usuários
├── Clientes (HUB)
│    ├── Acessos
│    ├── Certificados
│    ├── Documentos
│    ├── Extratos Bancários
│
├── Obrigações (transversal)
├── Notificações
├── Auditoria

```

---

## 4. Tipos de Entidade

### 4.1 Entidades centrais

- offices
- users
- clientes

---

### 4.2 Entidades dependentes do cliente

Dados que sempre pertencem a um cliente:

- acessos
- certificados
- documentos
- contas bancárias
- extratos

---

### 4.3 Entidades operacionais (transversais)

Dados que cruzam múltiplos clientes:

- obrigações
- notificações
- auditoria

## 10. Regras Arquiteturais

---

### 10.1 Isolamento

- todas as entidades devem conter `office_id` (direto ou indireto)

---

### 10.2 Relacionamento

- dados do cliente nunca devem existir sem cliente
- dados operacionais devem referenciar cliente

---

### 10.3 Consistência

- evitar duplicidade de dados
- evitar estruturas genéricas sem controle

---

## 11. Regra de Ouro

Se uma informação pertence a um cliente:

→ ela deve estar vinculada ao cliente

Se uma informação cruza múltiplos clientes:

→ ela deve ser tratada como módulo transversal
